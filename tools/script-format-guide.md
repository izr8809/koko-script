> 새 캐릭터 스크립트를 처음 작성할 때 이 문서만 보면 바로 시작할 수 있도록 정리한 레퍼런스.

### 파일 종류

각 캐릭터 폴더 안에는 토픽별 파일 5개가 들어간다:

| 파일 | 토픽 | 설명 |
|------|------|------|
| `CR.txt` | Career | 직장/알바 상황 |
| `KD.txt` | K-Drama | 드라마 장면 직접 연습 |
| `KP.txt` | K-Pop | 아이돌 팬 역할극 |
| `TR.txt` | Travel | 실제 장소 배경 여행 회화 |
| `PI.txt` | Personal Interest | 개인 관심사 대화 |

## 섹션 2: 파일 내부 구조

### 전체 구성

파일 하나 = **스토리 3개** (A0: 1~3번, A1: 4~6번)

스토리 하나 = **3sim × 2세트 = 6세트 = 12턴**

```
CR.txt
├── CR-1: [스토리 제목]
│   ├── sim1: [장면명]
│   │   ├── 세트 1 (기)
│   │   └── 세트 2 (승)
│   ├── sim2: [장면명]
│   │   ├── 세트 3 (승)
│   │   └── 세트 4 (전)
│   └── sim3: [장면명]
│       ├── 세트 5 (전)
│       └── 세트 6 (결)
├── CR-2: ...
└── CR-3: ...
```

### 기승전결 세트 구조

각 스토리는 기승전결 흐름을 따른다:

| 세트 | 위치 | 역할 |
|------|------|------|
| 세트 1 | sim1 | 기 (시작, 첫 만남) |
| 세트 2 | sim1 | 승 (전개 시작) |
| 세트 3 | sim2 | 승 (전개 심화) |
| 세트 4 | sim2 | 전 (전환점) |
| 세트 5 | sim3 | 전 (클라이맥스) |
| 세트 6 | sim3 | 결 (마무리) |

### sim 헤더 포맷 차이

**sim1은 `###` (H3), sim2/3은 `##` (H2)**

```
### sim1: 출근          ← H3
## sim2: 오전           ← H2
## sim3: 오후           ← H2
```

단, 세트 헤더는 반대로 sim1이 `**기 — 도착 [세트 1]**` (볼드), sim2/3이 `### 기 — 업무 [세트 1]` (H3)인 경우도 있다. 실제 파일을 보면 **CR-1은 sim1 세트가 볼드, CR-2/3은 H3**으로 혼용된다. 어느 쪽이든 일관성만 유지하면 된다.

### 세트 내부 구조

세트 하나 = **대사 2턴 + Outro**

```
**기 — 도착 [세트 1]**
User: 안녕하세요! 저 인턴이에요.
AI: …어. 왔어?
User: 네! 잘 부탁해요.
AI: 따라와.

**Outro:**
> He didn't smile. He didn't say "welcome."
> But he was there. Ten minutes early.
>
```

## 섹션 3: 대사 작성 규칙

### AI 대사 vs User 대사

- `AI:` 줄 = AI 캐릭터 대사
- `User:` 줄 = 학습자가 말할 대사

### `___` 처리 규칙

**User 대사에만 허용. AI 대사에는 절대 금지.**

User가 자기 이름을 말하는 상황에서만 사용:

```
User: ___이에요!          ← OK (이름 자리)
AI: …됐어.               ← OK

AI: ___야, 잘 왔어.       ← 절대 금지
```

### 이름 처리

`[이름]` 형식 금지. 이름이 들어갈 자리는 반드시 `___` 사용.

```
User: ___이에요.          ← OK
User: [이름]이에요.        ← 금지
```

### 괄호 주석 금지

대사 안에 괄호로 설명 넣는 것 금지:

```
AI: 잘 왔어. (반갑다는 뜻)    ← 금지
AI: 잘 왔어.                  ← OK
```

### 의성어/이모티콘 대사 직접 삽입 금지

`헤헤`, `ㅋㅋ`, `ㅠㅠ` 같은 표현은 대사에 직접 넣지 않는다. 캐릭터 특징 설명에는 언급할 수 있지만 실제 대사 줄에는 금지.

```
AI: 헤헤, 왔어?     ← 금지
AI: …왔어?          ← OK
```

### 영어 직역 패턴 금지

영어로 먼저 생각하고 번역하면 어색한 한국어가 나온다. 한국어로 먼저 생각할 것.

```
AI: 나는 너를 기다렸어.    ← 영어 직역 느낌
AI: 기다렸어.              ← 자연스러운 한국어
```

## 섹션 4: 토픽별 컨셉

### CR (Career)

직장 또는 알바 상황. **캐릭터마다 직장 배경이 다르다.**

- 시윤: 합정 디자인 에이전시 (선배)
- 새 캐릭터 작성 시 personas.md의 거주지/성격에 맞는 직장 배경 설정

AI 역할: 직장 동료, 선배, 상사 등 직장 내 관계

### KD (K-Drama)

AI가 드라마 상대역을 맡아 **드라마 장면을 직접 연습**하는 포맷.

- 드라마 감상 대화, 추천 대화 금지
- 실제 드라마 장면처럼 AI와 User가 역할극
- 로맨스, 갈등, 화해 등 감정선 있는 장면 구성

### KP (K-Pop)

AI가 아이돌 역할, User가 팬 역할을 맡는 역할극.

- 팬끼리 대화하는 포맷 금지 (단, KP-2 콘서트처럼 팬 동료 역할은 예외)
- 팬미팅, 사인회, 라이브, 콘서트 등 실제 팬 경험 장면
- AI가 아이돌 본인으로서 말함

### TR (Travel)

실제 장소를 배경으로 한 여행 회화.

**중요: 점원, 기사, 행인 등 서비스 역할의 AI는 반드시 존댓말 사용.**

```
AI: 사이즈는요?           ← 카페 점원 (존댓말)
AI: 타요.                ← 택시 기사 (반말도 가능, 실제 상황 반영)
```

장소 예시: 카페, 식당, 택시, 쇼핑몰, 길거리

## 섹션 5: 캐릭터별 말투 가이드

### 남성 캐릭터

| 캐릭터 | 아키타입 | 말투 | 먼저 말 거는 쪽 | 특징 |
|--------|----------|------|----------------|------|
| 도민 | 다정남 | 반말 | AI 먼저 | 에너지 넘침, 공감형 질문, 예측불가 한 방 |
| 시윤 | 츤데레 | 반말 | User 먼저 | 짧고 무뚝뚝, `…어.` / `딱히.`, 속마음은 따뜻함 |
| 한결 | 연하남 | 존댓말→반말 전환 | AI 먼저 | 골든리트리버 에너지, sim3에서 반말 전환 |
| 태오 | 4차원 | 반말 | AI 먼저 | 비유적 표현 (`파란 맛이야.` / `빵은 구름이야.`), 공감각적 세계관 |
| 현준 | 차도남 | ~요체 | User 먼저 | 짧고 정확한 배려, `…네.` / `다행이에요.` |

### 여성 캐릭터

| 캐릭터 | 아키타입 | 말투 | 먼저 말 거는 쪽 | 특징 | 대응 남성 |
|--------|----------|------|----------------|------|-----------|
| 민지 | 절친형 | 반말 | AI 먼저 | 수다스럽고 공감 잘 함, 리액션 큼 | 도민 |
| 서연 | 쿨언니 | 반말 | User 먼저 | 처음엔 무관심한 척, 은근히 챙겨줌 | 시윤 |
| 하늘 | 4차원 | 반말 | AI 먼저 | 예술가 감성, 뜬금없는 말이 왠지 맞음 | 태오 |
| 유나 | 귀여운 동생 | 존댓말→반말 전환 | AI 먼저 | 에너지 넘치고 밝음, 언니/오빠 잘 따름 | 한결 |
| 지아 | 커리어우먼 | ~요체 | User 먼저 | 바쁘고 단정함, 짧게 대답, 은근히 챙겨줌 | 현준 |

### 말투 적용 원칙

**"User 먼저" 캐릭터 (시윤, 현준, 서연, 지아)**
- sim1 세트 1은 User 대사로 시작
- AI가 먼저 말 거는 장면 금지

**"AI 먼저" 캐릭터 (도민, 한결, 태오, 민지, 하늘, 유나)**
- sim1 세트 1은 AI 대사로 시작 가능
- 단, 상황에 따라 User가 먼저 말하는 세트도 자연스럽게 섞임

**존댓말→반말 전환 캐릭터 (한결, 유나)**
- sim1~2: 존댓말
- sim3: 반말로 전환 (`나 반말해도 돼?` 같은 전환 장면 포함)

## 섹션 6: 난이도별 문장 길이

### A0 (Absolute Beginner)

- 2~5단어 짧은 문장
- 단답형 위주
- 복잡한 문법 구조 없음

```
AI: …어. 왔어?
User: 네! 잘 부탁해요.
AI: 따라와.
```

### A1 (Beginner)

- 5~10단어 조금 더 긴 문장
- 이유, 조건, 부탁 등 간단한 연결 표현 포함
- 자연스러운 구어체 유지

```
AI: …어, 왔어. 그냥 잘 해.
User: 오늘 날씨도 좋고 기분 좋아요.
AI: …그래? 별로 신경 안 썼어.
```

### A0 vs A1 스토리 번호

| 난이도 | 스토리 번호 | 예시 |
|--------|------------|------|
| A0 | 1~3 | CR-1, CR-2, CR-3 |
| A1 | 4~6 | CR-4, CR-5, CR-6 (같은 파일에 이어서) |

## 섹션 7: Outro 작성 가이드

### 기본 규칙

- **영어로 작성**
- **1~2줄**
- 대사가 아닌 **장면 묘사 또는 감정 서술**
- 이모지 사용 가능 (과하지 않게)
- `>` 인용 블록 형식 사용, 마지막 줄은 빈 `>` 로 닫기

### 포맷

```
**Outro:**
> [장면 묘사 또는 감정 서술]
> [두 번째 줄 (선택)]
>
```

### 좋은 Outro 예시

```
> He didn't smile. He didn't say "welcome."
> But he was there. Ten minutes early.
>

> "Not bad." From Siyun, that's a standing ovation. 🌟
> Next up: Sometimes the work runs past midnight.
>

> He said nothing. But he waited. 🌙
> Late nights here? Not so bad after all.
>
```

### Outro 작성 팁

- 캐릭터의 속마음이나 행동의 의미를 독자에게 살짝 귀띔하는 느낌
- 다음 sim이나 스토리로 이어지는 경우 `Next up:` 으로 예고 가능
- 너무 설명적이면 안 됨. 짧고 여운 있게

## 섹션 8: 실제 예시

시윤 CR-1 sim1 전체 예시:

```
### CR-1: 첫날이야

**대분류:** Career
**중분류:** Intern First Day
**난이도:** A0 (Absolute Beginner)
**구성:** 12턴 (6세트, 3sim)

---

### sim1: 출근

**📍 장소:** 합정 디자인 에이전시 (로비)

**Goal:** To arrive at the agency and meet your senior on your very first intern day.

**intro_text:**
First day at the Hapjeong design agency.
The lobby is quiet. But someone's already there.

**기 — 도착 [세트 1]**
User: 안녕하세요! 저 인턴이에요.
AI: …어. 왔어?
User: 네! 잘 부탁해요.
AI: 따라와.

**Outro:**
> He didn't smile. He didn't say "welcome."
> But he was there. Ten minutes early.
>

**승 — 자리 [세트 2]**
AI: 여기야.
User: 제 자리예요?
AI: 응.
User: 감사해요!

**Outro:**
> A one-word tour. That's all you got.
> The desk was clean. He'd cleared it himself.
>
```

### 포인트 정리

- sim1 헤더: `### sim1:` (H3)
- 세트 헤더: `**기 — 도착 [세트 1]**` (볼드)
- 장소: `**📍 장소:**` 형식
- Goal: 영어로 한 줄
- intro_text: 영어로 2~3줄, 장면 세팅
- 대사: `User:` / `AI:` 접두사
- Outro: `>` 블록, 영어, 1~2줄

## 섹션 9: 작성 전 체크리스트

파일 완성 전 아래 항목을 확인한다.

### 구조 체크

- [ ] 스토리 3개 (A0: 1~3, A1: 4~6)
- [ ] 각 스토리에 sim 3개
- [ ] 각 sim에 세트 2개
- [ ] 각 세트에 대사 4줄 (User 2, AI 2) + Outro
- [ ] 총 12턴 (세트 6개 × 2턴)
- [ ] sim1 헤더 `###`, sim2/3 헤더 `##`

### 대사 체크

- [ ] AI 대사에 `___` 없음
- [ ] `[이름]` 형식 없음 (모두 `___` 로 대체)
- [ ] 괄호 주석 없음
- [ ] `헤헤`, `ㅋㅋ` 등 의성어 대사 없음
- [ ] 영어 직역 느낌 없음

### 캐릭터 말투 체크

- [ ] 반말/존댓말 일관성 유지
- [ ] "User 먼저" 캐릭터는 sim1 세트 1이 User 대사로 시작
- [ ] 존댓말→반말 전환 캐릭터는 sim3에서 전환
- [ ] 캐릭터 특유의 말버릇 포함 (시윤: `…어.`, 태오: 비유 표현 등)

### 토픽별 체크

- [ ] **CR**: 캐릭터 직장 배경과 일치하는 장소/상황
- [ ] **KD**: 드라마 감상/추천 대화 없음, 장면 직접 연습
- [ ] **KP**: 팬 대화 포맷 (AI = 아이돌, User = 팬)
- [ ] **TR**: 서비스 역할 AI는 존댓말 사용

### Outro 체크

- [ ] 영어로 작성
- [ ] 1~2줄
- [ ] `>` 블록 형식, 마지막 빈 `>` 로 닫음
- [ ] 대사가 아닌 장면 묘사/감정 서술

## 빠른 참고: 시윤 말투 패턴

시윤은 가장 자주 쓰이는 레퍼런스 캐릭터. 말투 패턴 요약:

```
…어.          (기본 반응)
응.           (긍정)
됐어.         (완료/거절)
그냥 해봐.    (지시)
알아.         (인지)
…뭐, 나쁘지 않아.   (칭찬)
기다렸어.     (감정 표출, 드물게)
```

- 문장 앞 `…` 침묵 표시 자주 사용
- 짧을수록 진심에 가까움
- 행동으로 마음을 보여줌 (말로 하지 않음)
