# 한국어 네이티브 스크립트 작성 가이드

## 핵심 원칙

**영어로 생각하지 마. 한국어로만 생각해.**

에이전트가 영어로 먼저 생각하고 번역하면 이런 문제가 생김:
- `…시윤인데.` → "I'm Siyun, though." 직역
- `…취향은 있네.` → "You have taste." 직역
- `그건 인정.` → "Fair enough." 직역
- `한국어 하네.` → "You speak Korean." 직역

문법은 맞는데 네이티브가 실제로 안 쓰는 표현이 나옴.

---

## 프롬프트 작성 규칙

### 1. 언어 지시
프롬프트 맨 앞에 반드시 이 문장 넣기:

```
너는 지금 한국어로만 생각해야 해. 영어로 먼저 생각하고 번역하지 마.
각 대사를 쓰기 전에 "실제 한국 20대 [캐릭터 설명]가 이 상황에서 뭐라고 할까?" 를 먼저 떠올려.
```

### 2. 캐릭터 설명도 한국어로
- ❌ `tsundere, cold, minimal responses`
- ✅ `츤데레, 반말, 짧게 말함, 속마음과 반대로 행동`

### 3. 좋은 예시 / 나쁜 예시 한국어로 제시
```
대사 예시 (좋음): `…아. 그래.` / `없어.` / `…뭘.`
대사 예시 (나쁨): `반갑습니다!` / `취향은 있네.` / `그건 인정.`
```

### 4. 금지 패턴 명시
```
금지: 영어 직역 느낌 나는 표현
금지: 교과서 한국어 (반갑습니다, 저는 ~입니다)
금지: 괄호 주석 (웃으며), (머뭇거리며)
```

## 난이도별 문법 기준

### A0 (Absolute Beginner) — 단어 나열 수준
- 문장 길이: 2~5단어
- **허용**: ~야/이야, ~해, ~어, 있어/없어, 좋아/싫어, 단순 현재형
- **금지**: ~아/어 본 적 있다, ~긴 한데, ~는 건 아니고, ~잖아, ~을게요, ~아도 돼요, ~치고, ~고 싶다
- 어휘: 기초 단어만 (맛집 X → 식당 O)

### A1 (Beginner) — 간단한 문장 수준
- 문장 길이: 5~10단어
- **허용**: ~는데, ~면, ~잖아, ~고 싶다, ~ㄹ래, ~아/어 보다, ~을게
- **금지**: ~치고, ~긴 한데, ~는 건 아니고, ~아/어 본 적 있다
- 어휘: A0 + 일상 어휘 (맛집, 취향, 약속 등)

## 캐릭터별 말투 요약

| 캐릭터 | 말투 | 핵심 표현 | 금지 표현 |
|--------|------|-----------|-----------|
| 도민 (다정남) | 반말, AI 먼저 | `오 나도!` / `진짜?` / `같이 가자!` | 과한 존댓말, 딱딱한 표현 |
| 시윤 (츤데레) | 반말, User 먼저 | `…아.` / `딱히.` / `…언제?` | 친절한 표현, 감탄사 |
| 한결 (연하남) | 존댓말→반말, AI 먼저 | `진짜요? 대박!` / `반말 할래요?` | 무기력한 반응, 짧은 대답 |
| 태오 (4차원) | 반말, AI 먼저(엉뚱) | `파란 맛이야.` / `빵은 구름이야.` | 평범한 인사, 논리적인 대화 |
| 현준 (차도남) | ~요체, User 먼저 | `…네.` / `…그래요.` / `…좋아요.` | 감탄사, 긴 문장, 감정 표현 |

## 파일 포맷

```
1#ZP|세트1: 첫 인사, 이름 교환
2#MQ|AI 대사
3#VR|User 대사 (빈칸은 ___)
4#WS|AI 대사
5#NK|User 대사
6#TX|Narrator line in English.
```

- 세트당 4줄 대사 + 1줄 나레이션 = 6줄
- 6세트 × 6줄 = 36줄
- 해시: 2글자 대문자 랜덤 (ZP, MQ, VR 등)
- AI 대사에 `___` 절대 금지
- User 대사에만 `___`
- 괄호 주석 금지

## 저장 방법 (에이전트에게 항상 명시)

```python
content = """..."""
with open('/path/to/file.txt', 'w', encoding='utf-8') as f:
    f.write(content)
```

- Edit 툴 사용 금지
- 저장 후 반드시 `cat` 으로 확인
- "저장 완료"라고 해도 `git diff --stat` 으로 직접 검증
